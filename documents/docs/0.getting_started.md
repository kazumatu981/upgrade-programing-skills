## 序章

### 😰ソフトウェアを作るときの難しさ

#### ソフトウェアは難しい

- 形が見えない
- そもそも難しい
- 簡単に変わってしまう
- ユーザが使う環境に合わせる必要がある

#### 🤔ではどうする？

ソフトウェアを作るときは、ひとつひとつ形が見えるようにして、作るのが最大のポイント。

### 開発成功のキーワード

1. 可能な限り、大きな仕事を小さい仕事に分割して、
2. 常に、成果物が何をしてようとしているか<span class="red">「俯瞰的に」</span>見えるようにしておく

### ソフトウェア開発の大きな流れ

#### ソフトウェア開発工程

![ソフトウェア開発の流れ](./assets/figures/0.software_dvelop_flow.drawio.svg)

| 行程名 | 概要 |
| -- | -- |
| 要件定義 | 何をするためのソフトウェアかを決める |
| 設計 | どのような仕組み、どのような部品で構成するかを考える |
| 実装 | プログラミングをする |
| テスト | 期待通りに動作するかを確認する |


### ちゃんとしたプログラミングを書くコツ

* ちゃんと要件はまとまってますか？
* 設計ができていますか？
* コードが読みやすいですか？
* 何ができたら合格を決めてますか？
