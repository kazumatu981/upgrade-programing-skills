## 序章

### 😰ソフトウェアを作るときの難しさ

#### ソフトウェアは難しい

- 形が見えない
- そもそも難しい
- 簡単に変わってしまう
- ユーザが使う環境に合わせる必要がある

#### 🤔ではどうする？

ソフトウェアを作るときは、ひとつひとつ形が見えるようにして、作るのが最大のポイント。

### 開発成功の秘訣

1. 可能な限り、大きな仕事を<span class="red">小さい仕事に分割</span>
2. <span class="red">俯瞰的に見える</span>ようにしておく

![bg left h:70%](./assets/images/undraw_process_0wew.svg)

### ソフトウェア開発の大きな流れ

#### ソフトウェア開発工程

![ソフトウェア開発の流れ](./assets/figures/0.software_dvelop_flow.drawio.svg)

| 行程名 | 概要 |
| -- | -- |
| 要件定義 | 何をするためのソフトウェアかを決める |
| 設計 | どのような仕組み、どのような部品で構成するかを考える |
| 実装 | プログラミングをする |
| テスト | 期待通りに動作するかを確認する |

各工程を丁寧にすることが大事


### ソフトウェア開発を成功させるために

#### やるべきこと

* **要件定義:** ちゃんと要件はまとまってますか？
* **設計:** 設計の意図が伝わりますか？
* **実装:** コードが読みやすいですか？
* **テスト:** 何ができたら合格を決めてますか？

---

#### よくある誤解

![bg right h:50%](./assets/images/undraw_warning_tl76.svg)

* 設計しなくても、コードが書ける
* 自分は言語を知らないからコードが書けない
* テストなんてしなくてよい
* 仕様書が無くてもバグは見つかる
